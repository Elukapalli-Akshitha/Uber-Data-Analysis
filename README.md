# 🚖 Uber Demand-Supply Analysis

## 📌 Project Overview

This project focuses on analyzing Uber ride request data to identify patterns in demand and supply, detect key operational issues such as ride cancellations and unavailability of cars, and derive actionable business insights. The analysis was performed using a combination of Excel for data cleaning, Python (Pandas, Matplotlib, Seaborn) for exploratory data analysis, and MySQL for structured querying and insights generation.

## 🎯 Objective

The primary objective of this project is to understand:

* When and where ride demand is highest
* The major causes of trip cancellations
* Time periods with high “No Cars Available” scenarios
* Supply-demand gaps in different locations and time slots

## 🛠️ Tools & Technologies

* **Excel** – Data cleaning and preprocessing
* **Python (Pandas, NumPy, Matplotlib, Seaborn)** – Data analysis and visualization
* **MySQL** – Data querying and business insights extraction

## 🔍 Key Analysis Performed

* Data cleaning and preprocessing (handling missing values, formatting timestamps)
* Feature engineering (extracting hour and time slots)
* Univariate, bivariate, and multivariate analysis
* SQL-based aggregation and filtering for business insights

## 📊 Key Insights

* Peak demand occurs during morning and evening hours
* High cancellations are observed during specific time slots due to driver unavailability
* “No Cars Available” cases are more frequent during high-demand periods
* Airport pickup points show different demand patterns compared to city locations

## 💡 Business Recommendations

* Increase driver availability during peak hours
* Provide incentives to drivers for high-demand time slots
* Improve ride allocation strategies to reduce cancellations
* Balance supply across airport and city locations

## 📁 Project Files

* `data.csv` – Raw dataset
* `analysis.ipynb` – Python analysis and visualizations
* `uber_analysis.sql` – SQL queries for insights
